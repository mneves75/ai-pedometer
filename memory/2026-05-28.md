## Lessons

- The audit subagent's first pass had some imagined line numbers; always confirm with `grep -n` and re-read the actual files before patching. Several "P0 missing accessibility label" claims turned out to be false positives because the code already had `accessibleButton`/`.accessibilityLabel` upstream.
- Dynamic Type safety is cheap: switching `.frame(height: X)` to `.frame(minHeight: X)` on text-bearing cards/charts removes a whole class of clipping bugs at large accessibility sizes.
- Token centralization reduces the audit surface area for the next release; future drift only needs `grep -n "cornerRadius: [0-9]"` and `grep -n "\.frame(width: [0-9]"` to enforce.
- Always bump `project.yml` BEFORE running `xcodegen generate`. The generated `.xcodeproj` snapshots `project.yml` values, so a later edit to `project.yml` will not reach device/TestFlight builds until `xcodegen generate` is re-run.
- Resolve `DEVELOPMENT_TEAM` from `Config/Local.xcconfig` (canonical source) instead of parsing it out of `security find-identity` output, which can match the Apple Development identity's `(SM92FHMYS3)` (personal) when the project actually signs against `Q96FUTC5G8` (org).

## Lessons (fourth pass)

- **`@MainActor` is NOT a reentrancy guard for `async` methods.** It serializes only synchronous regions; every `await` releases the actor and lets another invocation interleave. An `async` method with shared mutable state and multiple concurrent callers needs an explicit in-flight/serialization guard. A prior cycle's "idempotent under @MainActor" justification was wrong for exactly this reason.
- **Prove a race test has teeth.** A passing concurrency test is meaningless unless it fails without the fix. Add a concurrency counter to the mock (max concurrent entries), bypass the guard once, confirm the failure, then restore.
- **A "no user-visible effect" justification must trace every consumer.** The badge duplicate-row dedup was correct for the Badges UI but missed the AI prompt consumer (`fetchEarnedBadgeCount` → `totalBadgesEarned`). Follow the value to *all* surfaces before declaring it invisible.
- **XCUITest `tab_more`-style "element not found" failures on a loaded machine are usually launch-timing flakes** — confirm by isolated re-run before treating as a regression; don't bump or block on a single full-suite UI flake that passes alone.
