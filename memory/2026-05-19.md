## Findings shipped

- **P1 — Heart-rate clobber on Motion fallback.** `applyMotionSnapshot` was unconditionally setting `todayHeartRateBPM = nil`. Fix introduced a `HeartRateRefreshAttempt(attempted: Bool, value: Double?)` tri-state so the Motion fallback path always re-fetches HR independently of steps, and only mutates the published value when the fetch actually ran. Cycle 2 refinement: when sync is enabled we always overwrite, including with `nil`, to match Apple’s own “fade to no data” behavior.
- **P1 — `GPXRouteParserError` had no `LocalizedError`.** The file-importer alert in `WorkoutsView` was showing the system fallback “The operation couldn’t be completed (… error 0.)”. Now every case has a localized en + pt-BR string and the alert reads as intended.
- **P1 — Cold-launch lifecycle race.** `AppLifecycleCoordinator.handle(.active)` stamped `lastPhase` before checking `isStartupComplete()`, so the only `.active` event SwiftUI fires at launch was swallowed if startup was slow. Fix: only commit `lastPhase` on the success path for `.active`; non-active transitions still commit immediately. Regression test reproduces the cold-launch sequence and proves the foreground work fires once startup catches up.
- **P1 — Cancelled startup latches `didStart`.** `AppStartupCoordinator.startIfNeeded` set the flag before any await, so a cancelled task left the app permanently half-initialized. Fix: when `Task.isCancelled` is true at the function tail, reset `didStart` so the next invocation can pick startup back up. Verified sub-steps are idempotent (BG task registration, motion live updates, sync version-gated cold start).
- **P1 — Widget reload throttle suppresses the post-midnight refresh.** The 5-minute + 200-step throttle was blocking the first reload after day rollover (steps reset to 0, delta to yesterday’s total is negative). Extracted a pure `shouldReloadWidgets` helper that releases on calendar-day change; four regression tests cover throttle on / off / midnight / first-call.
- **P1 — Dashboard / Settings / AICoach / History `#Preview` blocks omitted `PremiumAccessStore`.** Each one reads it from the environment; the previews would crash at runtime. Injected `PremiumAccessStore(forcedPremiumEnabled: true, isTesting: true)`.
- **P2 — WatchConnectivity `sendMessage` fanout.** `WatchSyncService.send` was firing on every CMPedometer tick. Added a static `shouldSendReachableMessage` helper with a 5 s / 10 step coalescing window; `updateApplicationContext` still runs every tick because the system coalesces that channel.
- **INFO — Wheelchair distance.** Both `HealthKitService.fetchDailySummaries` and `StepTrackingService.resolveDistance` hardcode `0` for wheelchair pushes even though Apple ships `HKQuantityTypeIdentifier.distanceWheelchair`. Out of scope for an audit — requires entitlement + privacy manifest review — logged as Q1 in `implementation-notes.html`.

The dashboard appends a <code>RelativeDateTimeFormatter</code> caption (“· 12m ago”) whenever the sample is ≥ 5 minutes old.
