## Landmines de ambiente desta máquina (2026-06-13)

- **GitHub push**: o `gh`/git resolve credencial para a conta `conhecendoia` por padrão, que NÃO
  tem write em `mneves75/ai-pedometer` → 403. Fix: `gh auth switch --user mneves75` antes de push
  (ambas as contas estão no keyring; mneves75 é o dono). Protocolo git do gh é ssh.
- **Runtime watchOS 26.x foi REMOVIDO** desta máquina (só resta watchOS 27.0) e a Apple NÃO o
  disponibiliza para download (26.5/26.4/26.0 → "not available for download"). O `actool` do watch
  para **simulador** exige device de thinning Series 7 (45mm) na era 26.5, incompatível com 27.0 →
  **build de SIMULADOR do scheme falha** ("Failed to find a suitable device... Series 7 (45mm)...
  watchOS 27.0"). `runtime match set watchos26.5 → 27.0` redireciona o SDK mas não muda a era do
  device. WATCHOS_DEPLOYMENT_TARGET=27.0 não ajuda (SDK continua 26.5). Workaround para verificar
  iOS: remover TEMPORARIAMENTE `- target: AIPedometerWatch` das deps do app em `project.yml`,
  `xcodegen generate`, buildar, restaurar. NUNCA commitar essa remoção (o autoreview pega como P2).
  Build de DEVICE/archive (`--platform watchos`, físico) pode contornar pois não usa runtime de sim
  — a verificar no archive.
- **CoreSimulator instável + DISPUTADO**: há uma sessão paralela nesta máquina rodando OUTRO app
  ("Paquera"/"Seu parceiro de IA", um app de paquera com IA — projeto diferente) que cria/boota
  simuladores próprios (SV-Redesign, AIR-Redesign, CV-TestRunner, AIR-RD3) e DERRUBA meus sims
  (SIGKILL 137 em test runs; foreground roubado). `killall -9 CoreSimulatorService` deixa sessões
  fantasma no SimLaunchHost ("Bad or unknown session") — EVITAR; preferir `simctl shutdown all`.
  Para verificação visual confiável: criar um sim dedicado de nome único e capturar via
  `describe` (ax-service do argent funciona) + `axe tap` + `simctl io screenshot`. O `launch-app`/
  `screenshot`/`gesture-*` do argent 0.7.0 falham nesta máquina (simulator-server não sobe / injeção
  DYLD falha); `describe` funciona. Argent 0.11.0 disponível.
- **Demo mode** (`demo_mode_fake_data`) é lido do **app group** `group.com.mneves.aipedometer`
